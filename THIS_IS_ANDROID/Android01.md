# 📱코틀린 안드로이드 이해하기
## 운영체제와 플랫폼
안드로이드 플랫폼은 리눅스 운영체제 안에서  
리눅스와 상호작용하면서 동작한다.  
  
이미지 추가 예정
  
* System Apps
* Java API Framework
* Native C/C++ Libraries, Android RunTime
* Hardware Abstraction Layer
* Linux Kernel
  
## 플랫폼 버전과 API 레벨
안드로이드는 플랫폼 버전과 별개로 API 레벨이 존재한다.  
* API: 개발자에게 플랫폼의 기능을 제공해주는 도구  

API 레벨에 따라 변경사항이나 새로운 기능이 추가되는 경우가 존재한다.
API 레벨이 변경됨에 따라 기존 코드를 확인할 필요가 있다.  
현재 사용현황에 따라서 프로젝트의 API 레벨을 조절할 것
  
## 코틀린
과거 안드로이드 플랫폼에서는 JVM에서 앱이 동작했다.  
현재는 JVM을 대신해 Android RunTime을 사용한다.  
**JVM이 아니지만,  가상머신 위에서 앱이 동작한다는 점에서 자바의 동작구조를 차용하고있다.**  
기존에는 자바를 주로 앱 개발 언어로 사용했지만,  
2017년부터 코틀린을 공식 언어로 채택한 후 코틀린의 사용빈도가 높아지고 있다.
구글은 **코틀린 퍼스트**를 선언하고, 모든 문서에서 코틀린 코드를 먼저 노출하는등 코틀린을 지원하고 있다.
  
코틀린은 함수형 프로그래밍을 표방하고 있다.  
  
### 안드로이드 개발에 있어서 코틀린과 자바의 차이
코틀린은 함수형 프로그래밍 언어지만, 자바와의 호완을 위해  
객체지향 언어처럼 클래스내에서 함수를 작성해야한다.  
그럼에도 코틀린으로 작성했을때 코드가 간결해지는 효과를 볼 수 있다.
